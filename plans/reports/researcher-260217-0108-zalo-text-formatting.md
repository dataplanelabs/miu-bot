# Zalo Text Formatting Research Report

**Date:** February 17, 2026
**Research Scope:** Text formatting capabilities in Zalo messenger
**Status:** Complete

---

## Executive Summary

Zalo **does NOT support Markdown syntax**, but offers **native formatting controls** via its UI editor (primarily on PC version). The app supports basic text styling through built-in buttons, Unicode workarounds for styled text, and structural formatting like lists and bullet points.

---

## Key Findings

### 1. Native Text Formatting (PC/Desktop Version)

**SUPPORTED:**
- **Bold** (B button) - Full support via UI editor
- **Italic** (I button) - Full support via UI editor
- **Underline** (U button) - Full support via UI editor
- **Strikethrough** (Strike button) - Full support via UI editor

**How it works:** Select text → Click formatting button → Formatting persists in sent message

Mobile versions (Android/iOS) have limited formatting support compared to PC.

### 2. Lists and Bullet Points

**SUPPORTED:**
- **Bullet point lists** - Via UI editor on PC
- **Numbered/ordered lists** - Via UI editor on PC
- **Indentation** - Increase/decrease indent buttons available

**How it works:**
1. Select or type text
2. Click bullet (•) or numbered list icon
3. Use indent buttons to adjust nesting
4. Formatting persists in sent message

**Limitation:** Mobile versions do NOT have visual list support in UI.

### 3. Text Size Formatting

**SUPPORTED:**
- **Size adjustment** - Via "2 A's" icon (increase/decrease font size)
- Multiple size presets available

## Unicode Character Workarounds

While native formatting is preferred, users can leverage Unicode for cross-platform consistency:

### Bold/Italic Unicode Alternatives
- Bold: Copy bold Unicode text from generators (e.g., Yaytext) → Paste into Zalo
- Italic: Copy italic Unicode text from generators → Paste into Zalo
- Works on both mobile and PC versions
- Preserves styling when sent

### Zalgo Text (NOT Recommended)
- Generated by overlaying Unicode combining marks (U+0300–U+036F)
- Creates glitchy-looking text with marks above/below/through characters
- Works on Zalo but has issues:
  - Reduced readability
  - May cause lag on mobile clients
  - Inconsistent rendering across devices
  - Not suitable for business/professional use

## Zalo-Specific Formatting Features

### Official API Support (zlapi)
The unofficial Python Zalo API supports formatting styles:
- `"font"` - Font type
- `"bold"` - Bold text
- `"italic"` - Italic text
- `"underline"` - Underlined text
- `"strike"` - Strikethrough
- `"color"` - Text color (hex format, e.g., "ffffff")

**Note:** Official Zalo APIs have limited markdown documentation; formatting may require custom implementation.

### Message Synchronization
- Formatted messages from Android/iOS phones sync to PC version
- PC editor provides full formatting UI
- Formatting persists cross-device after version 20.04.02

## Best Practices for Structured Text in Zalo

### What Works Best
1. **Use native UI formatting** on PC for professional messages (bold, underline for emphasis)
2. **Create lists** for structured information (bullet points or numbered)
3. **Line breaks** - Use Alt+Enter to create visual separation
4. **Unicode bold/italic** for mobile-first communications (copy from text generators)

### What to Avoid
1. Markdown syntax (`**bold**`, `*italic*`) - NOT parsed
2. HTML tags - NOT parsed
3. Zalgo text - Poor readability and platform compatibility
4. Excessive formatting - Difficult to read with too many styles

### Structured Text Template Example (Using Linebreaks + Lists)

```
IMPORTANT: Title of Message
━━━━━━━━━━━━━━━━━━━━━

• Item 1: Description
• Item 2: Description
• Item 3: Description

Alternative: Use numbered list for sequences
```

## Implementation Recommendations

### For Nanobot Zalo Channel

If sending formatted messages via bot:

1. **For PC Users:** Use native formatting where possible
   - Supported characters/emojis for structure
   - Line breaks for separation
   - Simple emphasis (bold key items)

2. **For Mobile Users:** Prioritize plain text with:
   - Emoji separators (📌, ⚠️, ✅)
   - Line breaks
   - ALL CAPS for headings
   - Unicode formatting as fallback

3. **Recommendation:** Default to **plain text + smart emoji use** for maximum compatibility
